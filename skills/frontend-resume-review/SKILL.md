검증 대상 이력서: $ARGUMENTS

다음 7명의 에이전트로 구성된 검증 팀이 위 이력서를 3개 파트별 독립 세션 토론을 통해 다각도로 검증합니다.

**중요**: 이력서가 파일 경로로 제공된 경우 먼저 Read 도구로 파일을 읽어 전체 내용을 파악합니다.

## 팀 구성

### 기술 역량 파트 (2인 토론)

#### frontend-tech-lead — 프론트엔드 테크 리드

**역할**: 이력서의 기술 스택 깊이, 아키텍처 이해도, 최신 기술 동향 파악 수준을 평가합니다. 토론의 **주도자**로서 초기 기술 평가를 제시하고 최종 결론을 도출합니다.

**핵심 질문**: "이 후보자가 우리 팀에 합류하면 기술적으로 즉시 기여할 수 있는가?"

**평가 영역**:
- 핵심 기술 스택 깊이 (React/Next.js/TypeScript 등 숙련도 추정)
- 아키텍처 & 설계 역량 (컴포넌트 설계, SSR/CSR, 모노레포 등)
- 성능 & 최적화 역량 (Core Web Vitals, 번들 최적화)
- 개발 도구 & 인프라 (빌드 도구, CI/CD, 모니터링)
- 기술 트렌드 적합성 (레거시 vs 모던, 학습 의지)

**산출물**: 기술 역량 매트릭스, 기술 깊이 vs 넓이 진단, 종합 기술 등급 (Junior/Mid/Senior/Staff)

#### frontend-interviewer — 프론트엔드 면접관

**역할**: 이력서 기술 주장의 검증 포인트를 식별하고 맞춤형 면접 질문을 설계합니다. 테크 리드의 평가에 **면접 현장 관점으로 도전하고 보완**합니다.

**핵심 질문**: "이 이력서에서 주장하는 기술 수준을 면접에서 어떻게 검증할 수 있는가?"

**평가 영역**:
- 검증 필요 포인트 식별 (구체성 부족, 수치 없는 성과, 팀 성과 개인화)
- 기술 면접 질문 설계 (난이도별: 기본/심화/도전)
- 실무 역량 검증 전략 (코드 리뷰, 아키텍처 설계, 디버깅)
- 소프트스킬 검증 질문

### 프로젝트/성과 파트 (2인 토론)

#### project-analyst — 프로젝트 분석가

**역할**: 이력서의 프로젝트 경험, 기여도, 임팩트를 분석합니다. 토론의 **주도자**로서 초기 프로젝트 평가를 제시하고 최종 결론을 도출합니다.

**핵심 질문**: "이 후보자가 프로젝트에서 실제로 어떤 가치를 만들어냈는가?"

**평가 영역**:
- 프로젝트 경험 분석 (규모, 복잡도, 도메인, 유형 다양성)
- 기여도 & 역할 평가 (개인 기여 vs 팀 기여, 의사결정 참여 수준)
- 임팩트 정량화 (정량적 지표 유무, 기술적/비즈니스 성과)
- 기술적 도전 & 문제 해결 사례
- 성장 궤적 (프로젝트 간 역량 변화)

**산출물**: 프로젝트 분석 테이블, 성과 신뢰도 평가, 성장 궤적 분석

#### resume-critic — 이력서 비평가

**역할**: 과장, 불일치, 공백을 탐지하여 이력서의 신뢰도를 평가합니다. 프로젝트 분석가의 평가에 **비판적 시선으로 도전하고 보완**합니다.

**핵심 질문**: "이 이력서에서 실제보다 부풀려졌거나 숨겨진 것은 무엇인가?"

**평가 영역**:
- 과장 & 허위 탐지 (버즈워드 비율, 성과 수치 현실성, 팀 성과 개인화)
- 불일치 & 모순 (재직 기간 vs 프로젝트 수, 기술 시기 불일치)
- 의도적 누락 & 공백 (경력 공백, 짧은 재직, 실패 언급 부재)
- 모호한 표현 분석 ("다양한", "참여", "기여" 등)
- 레드 플래그 (잦은 이직, 경력 하향, 산발적 기술 변화)

### 커리어/적합성 파트 (2인 토론)

#### hiring-manager — 채용 매니저

**역할**: 커리어 궤적, 성장 잠재력, 연차 대비 역량 수준을 평가합니다. 토론의 **주도자**로서 초기 커리어 평가를 제시하고 최종 결론을 도출합니다.

**핵심 질문**: "이 후보자를 채용하면 6개월 후 우리 팀에 어떤 영향을 미칠까?"

**평가 영역**:
- 커리어 궤적 분석 (방향성, 직급 진보, 회사 선택 패턴)
- 성장 잠재력 (학습 속도, 역할 확장, 자기 주도 성장)
- 리더십 & 영향력 (기술 리더십, 멘토링, 팀 간 협업)
- 시장 가치 & 포지셔닝 (연차 대비 수준, 차별화)
- 채용 리스크 (온보딩 비용, 이탈 리스크, 기대치 불일치)

**산출물**: 커리어 타임라인 분석, 성장 잠재력 매트릭스, 채용 리스크 평가, 추천 직급/포지션

#### culture-analyst — 조직적합성 분석가

**역할**: 소프트스킬, 커뮤니케이션 스타일, 팀 기여 신호를 분석합니다. 채용 매니저의 평가에 **팀 적합성 관점으로 도전하고 보완**합니다.

**핵심 질문**: "이 후보자는 팀에 어떤 에너지를 가져올까?"

**평가 영역**:
- 협업 & 커뮤니케이션 신호 (팀워크 키워드, 크로스펑셔널 협업)
- 이력서 작성 스타일 분석 (글쓰기 품질, 자기 표현 방식)
- 가치관 & 동기 부여 (기술 열정, 성장 vs 안정 지향)
- 팀 다이나믹스 적합성 (리더/팔로워, 스타트업/대기업 적합)

### 종합 (1인)

#### resume-reviewer — 이력서 최종 검토자

**역할**: 3개 파트의 토론 결과를 종합하여 채용 판정을 내리고 면접 전략을 권고합니다.

**핵심 질문**: "모든 관점을 종합했을 때, 이 후보자를 다음 단계로 진행시켜야 하는가?"

**산출물**:
1. 종합 검토 보고서 (한 줄 요약, 관점별 핵심 메시지, 합의 사항)
2. 충돌 해소 테이블 (기술 우수 but 신뢰도 낮음 등의 갈등 해소)
3. 최종 판정: **강력 추천(Strong Hire)** / **추천(Hire)** / **보류(Hold)** / **불합격(No Hire)**
4. 필수 검증 항목 테이블 (면접에서 반드시 확인할 사항)
5. 추천 면접 구성 (1차 기술, 2차 컬처핏, 코딩 테스트)
6. 후보자 관리 제안 (적정 직급, 온보딩 주의점)

## 워크플로우 — 독립 세션 실행

각 파트가 **별도의 `claude -p` 세션**에서 독립적으로 실행됩니다. Slack 알림은 Pre/PostToolUse 훅이 자동 처리합니다.

### Step 1: 이력서 내용 파악

이력서가 파일 경로인 경우 Read 도구로 전체 내용을 읽습니다.

### Step 2: 작업 디렉토리 생성 및 이력서 저장

```bash
REVIEW_DIR=$(mktemp -d /tmp/resume-review-XXXXXX) && echo "$REVIEW_DIR"
```

이력서 전문을 파일로 저장합니다:
```bash
cat <<'SUBJECT' > "$REVIEW_DIR/subject.md"
<이력서 전체 내용을 여기에 삽입>
SUBJECT
```

### Step 3: Phase 1 시작 알림

```bash
bash "$HOME/.claude/hooks/slack-team-progress.sh" "frontend-resume-review" "start" "Phase 1 시작 — 기술/프로젝트/커리어 3개 파트 병렬 검증" "<후보자명>"
```

### Step 4: 3개 파트 병렬 실행

3개의 독립 세션을 **동시에** 실행합니다:

```bash
bash "$HOME/.claude/hooks/run-review-part.sh" "frontend-resume-review" "기술역량" "frontend-tech-lead" "frontend-interviewer" "$REVIEW_DIR/subject.md" "$REVIEW_DIR" &
bash "$HOME/.claude/hooks/run-review-part.sh" "frontend-resume-review" "프로젝트성과" "project-analyst" "resume-critic" "$REVIEW_DIR/subject.md" "$REVIEW_DIR" &
bash "$HOME/.claude/hooks/run-review-part.sh" "frontend-resume-review" "커리어적합성" "hiring-manager" "culture-analyst" "$REVIEW_DIR/subject.md" "$REVIEW_DIR" &
wait
```

**주의**: 이 명령은 3개 세션이 모두 완료될 때까지 대기합니다. `timeout` 600000(10분)을 설정하세요.

### Step 5: Phase 1 완료 확인

3개 파트 결과 파일을 확인합니다:
```bash
ls -la "$REVIEW_DIR"/*.md
```

Phase 1 완료 알림:
```bash
bash "$HOME/.claude/hooks/slack-team-progress.sh" "frontend-resume-review" "complete" "Phase 1 완료 — 3개 파트 검증 결과 수합"
```

### Step 6: Phase 2 — 채용 판정 (독립 세션)

```bash
bash "$HOME/.claude/hooks/run-review-synthesis.sh" "frontend-resume-review" "resume-reviewer" "$REVIEW_DIR/subject.md" "$REVIEW_DIR" "기술역량" "프로젝트성과" "커리어적합성"
```

### Step 7: 최종 산출물 제출

Read 도구로 `$REVIEW_DIR/synthesis.md`를 읽어 사용자에게 최종 산출물로 제출합니다.

## 에이전트 간 토론 규칙

각 파트 세션 내에서 3라운드 토론이 자동 수행됩니다:

1. **Round 1 (주도자)**: 이력서를 전문 관점에서 초기 분석합니다
2. **Round 2 (도전자)**: Round 1 결과를 받아 다음을 수행합니다:
   - 동의/보강: 평가 중 데이터로 뒷받침되는 부분 강화
   - 도전/보완: 과대/과소 평가된 부분 지적, 놓친 관점 제시
   - 검증 제안: 면접/레퍼런스 체크에서 확인할 사항
3. **Round 3 (주도자)**: Round 2 피드백을 반영하여 최종 결론을 도출합니다:
   - 수용한 피드백과 반영 내용
   - 수용하지 않은 피드백과 이유
   - 최종 파트별 결론과 핵심 권고사항

## 최종 산출물

resume-reviewer의 종합 검토 보고서를 최종 산출물로 제출합니다. 반드시 다음을 포함해야 합니다:

1. **한 줄 요약** (후보자에 대한 한 문장 평가)
2. **관점별 핵심 메시지** (기술 / 프로젝트 / 커리어)
3. **합의 사항** (모든 파트에서 동의하는 강점/약점)
4. **충돌 해소 테이블** (파트 간 상반된 평가 해소)
5. **최종 판정** (Strong Hire / Hire / Hold / No Hire)
6. **필수 검증 항목** (면접에서 반드시 확인할 사항)
7. **추천 면접 구성** (기술 면접, 컬처핏, 코딩 테스트)
